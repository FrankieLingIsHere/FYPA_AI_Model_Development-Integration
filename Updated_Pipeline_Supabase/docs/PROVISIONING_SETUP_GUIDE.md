# Zero-Touch Edge Node Provisioning - Setup Guide

This guide outlines exactly how to configure the Cloud Dashboard so you can reliably deploy physical hardware to your workers on site with **Zero-Touch Configuration**.

## Step 1: Cloud Dashboard Setup

Before any local `start.bat` node can verify itself, you must ensure your Cloud Dashboard (Railway or Supabase deployments) is configured securely to receive them.

In your Cloud Vendor's Settings Panel, add the following Environment Variables (`.env`):

### 🔐 Security Requirements (Mandatory)
- **`ADMIN_PASSWORD`**: Set a strong, secure password here. (e.g., `SuperSecureHQA123!`). 
  > *This prevents unauthorized users from approving edge devices even if they discover your Admin Portal.*

### 📧 Email Notification Webhook (Strongly Recommended)
By configuring the SMTP credentials below, you receive **Instant Email Alerts** with a `Magic Approval Link` the exact second a worker plugs in a new Edge node on site.

- **`SMTP_SERVER`**: The outgoing server map. (e.g., `smtp.office365.com` or `smtp.gmail.com`)
- **`SMTP_PORT`**: Built-in ports. Set to `587` to leverage TLS (recommended).
- **`SMTP_USERNAME`**: The sender email address (e.g., `luna-system-admin@outlook.com`).
- **`SMTP_PASSWORD`**: Use an **App Password** generated by your email provider. Do NOT use your primary unencrypted login password.
- **`ADMIN_EMAIL`**: The email address of the person responsible for approving new nodes (e.g., `site_lead@yourcompany.com`). You will get the approval requests here.

*(Optional Alternative: If you just want Discord/Slack, fill `NOTIFICATION_WEBHOOK_URL` in directly and skip SMTP).*

## Step 2: Preparing the Local Payload 

Now that your Cloud Dashboard is actively protecting the border, prep the Edge Machine!

1. Move the `start.bat` installer suite onto the USB or Edge Node.
2. Edit **line 23** of `start.bat` using a basic text editor.
3. Change `set CLOUD_URL=https://YOUR_DASHBOARD_URL_HERE` to exactly match your public UI domain. Example:
   ```bat
   set CLOUD_URL=https://luna-ppe-dashboard.up.railway.app
   ```
4. Save the `.bat` file.

**That is all! No credentials whatsoever are pushed to the physical drive.**

## Step 3: Zero-Touch Deployment Floor Check

1. The Worker executes `start.bat`.
2. The UI pauses: `"Waiting for admin to approve Machine ID: Edge-ABCD"`
3. Check your `ADMIN_EMAIL` Inbox! You will see an alert with a secure "Magic Approval URL".
4. **Click the URL.** (Your browser will rapidly request the `ADMIN_PASSWORD` you set in Step 1).
5. Upon correct entry: The Edge Node instantly downloads your `SUPABASE_DB_URL` secrets behind the scenes, constructs its `.env` independently, closes the waiting screen, and fully boots into the tracking module!
