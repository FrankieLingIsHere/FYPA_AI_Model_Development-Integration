    Safety Manual

    Everything you need to operate the CASM PPE Safety Monitor

    Welcome

    Welcome to the CASM PPE Safety Monitor Manual Handbook. This handbook explains how to operate the safety monitoring system and answers common questions.

    Usage Stages

    Move through the stages below when setting up a monitoring session, reviewing evidence, or preparing a PC for local mode.

    Preliminary Hardware

    • Use a Windows PC with a webcam or RealSense camera connected before starting the local backend.
    • Keep stable Wi-Fi available for cloud sync, first-time provisioning, and report upload.
    • For local AI mode, install Ollama and keep the configured local model available before relying on offline generation.
    • Allow browser camera permission when using browser-camera monitoring.

    Live Monitoring Flow

    1. Open Live, choose the camera source, then confirm the preview is visible.
    2. Click Start and keep the worker area inside the camera frame.
    3. Watch for bounding boxes, voice alerts, and violation notifications.
    4. Click Stop before switching camera source or closing the session.

    Reading Reports

    • Open Reports to compare status badges: Queued, Generating, Ready, Failed, or Local Synced.
    • Use Open Report for completed reports, or Process/Reprocess Now when a report needs recovery.
    • Check the source badge before opening: Cloud uses the cloud backend, Local uses local files, and Local Synced is a local-origin report uploaded to cloud storage.
    • Cached cloud reports can open from browser storage; newly cached reports include embedded evidence images for local viewing.

    Local Mode Setup

    1. On the host PC, run Local Mode Checkup from Settings to detect machine ID, backend health, camera readiness, and model readiness.
    2. If approval is needed, use Request Provisioning and wait for admin approval in the cloud device portal.
    3. After approval, download or re-issue the installer BAT only from the approved machine.
    4. When Wi-Fi drops, CASM can continue locally; when Wi-Fi returns, synced-local reports queue back to Supabase automatically.

        PPE Guidelines

        • High-visibility safety vest required
        • Safety boots for floor work
        • Respirator mask in dusty / hazardous environments

        Live Monitoring

        The live monitoring feature performs real-time PPE detection using your camera and the YOLO model. Bounding boxes are drawn around detected PPE items, and violations are logged automatically.

        Instructions

        To start live monitoring, open the Live page and click the Start button.

        1. Click Start to begin live monitoring
        2. Your camera will activate and show the live feed
        3. YOLO detects PPE in real-time with bounding boxes
        4. Detected violations are logged automatically
        5. Click Stop to end the monitoring session

        Voice Alerts

        Voice alerts notify users audibly when PPE violations are detected. Toggle them on or off using the Voice Alerts button in the sidebar, and use Test Voice to verify your output device.

        Notification Center

        Every toast notification is now persisted to a private history. Open it from the floating bell at the bottom-left of the screen.

        • Unread badge shows how many alerts you have not yet seen.
        • Categories: Detection, Report, Sync, Admin, System.
        • Priorities: high (errors / violations), medium (warnings), low (info).
        • Mark all read clears the badge; Clear empties the history.
        • History is stored locally in your browser only. Nothing is uploaded.

        Admin & Device Provisioning

        The cloud admin portal at /admin/devices (Basic Auth protected) lets approved staff manage local-mode hardware.

        Actions

        • Approve a pending device to allow bootstrap exchange.
        • Reject removes a pending request.
        • Revoke Access (new) immediately invalidates an already-provisioned device. The device's stored credentials become useless and it must re-request approval.
        • Issue One-Time Installer Download generates a single-use installer signed for the target machine.
        • View Audit Log shows the append-only history of every request, approval, rejection, revocation, and provisioning event with timestamps and actor.

        Security

        • Provision secrets are stored hashed; raw secrets never leave the device on subsequent requests.
        • Bootstrap tokens are single-use (JTI replay protection).
        • Each request captures IP and user-agent into the audit log.

        Frequently Asked Questions

        Can I disable voice alerts?

        Yes. Toggle voice alerts using the button in the sidebar.

        What PPE does the system detect?

        The system detects helmets, safety vests, boots, and masks.

        Where are reports stored?

        Reports are stored in Supabase and accessible from the Reports page.
