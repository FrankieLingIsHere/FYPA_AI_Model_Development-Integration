# Worklog - 2026-05-04

## Summary of Work
Today's efforts focused on stabilizing the CASM Local Mode, specifically addressing regressions in notification latency, dashboard metric accuracy, and redundant report reprocessing during mode transitions.

## Key Accomplishments

### 1. Local Mode Stability & Robustness
- **[FIXED] Reprocessing Bug**: Resolved an issue where previously synced or completed local reports were being re-processed by the recovery sweep. Added a DB status check and a `SYNCED.txt` marker system to ensure each violation is processed exactly once.
- **[FIXED] Duplicate Processing**: Improved the recovery sweep to check if a report is already `generating` or `processing` in the database, preventing race conditions where multiple workers hit the local Ollama backend simultaneously.
- **[OPTIMIZED] Ollama Concurrency**: Increased the report generation wall-clock timeout from 180s to 300s to better accommodate the high-latency nature of local LLM inference (Gemma 3 4B) on consumer-grade hardware.

### 2. Performance & Latency Improvements
- **[FIXED] Notification Latency**: Optimized `process_violation` in `casm_app.py` to save the original image and preliminary `metadata.json` immediately upon detection. This ensures the frontend detects the new violation in real-time before the long-running NLP generation begins.
- **[FIXED] SSE Responsiveness**: Reduced the backend SSE heartbeat/update sleep interval from 2s to 1s for snappier UI updates.

### 3. Frontend & Dashboard Enhancements
- **[FIXED] Dashboard Metric Accuracy**: Updated `api.js` to recalculate total violation counts and category breakdowns from the merged cloud+local dataset. This ensures the dashboard remains accurate when switching to Local Mode.
- **[FIXED] Automatic Refresh**: Added network state listeners (`online`/`offline`) to `realtime.js` to automatically reconnect the SSE/Websocket stream and refresh the reports list when the connection mode changes.

### 4. General Fixes
- **[FIXED] Installer Download**: Restored installer download tokens to one-time consumption and removed stale "Reusable for 10m" admin labels so the UI matches the secure provisioning contract.
- **[FIXED] Session Clock Drift**: Added a 5s grace period to the violation monitor to prevent false "stale session" alerts caused by minor clock desync.

### 5. Follow-up Hardening: Sync, Notifications, and Low-Egress Tests
- **[FIXED] Local Cache Sync Contract**: Updated `/api/reports/sync-local-cache` dry-run handling so a missing queue manager no longer returns `503 Queue is not initialized`; dry-run now returns the expected counters without enqueueing work.
- **[FIXED] Recovery Queue Self-Heal**: Updated recovery execution to lazily initialize the queue manager before queue-dependent checks, preserving real worker-health errors for non-dry-run operations.
- **[FIXED] Duplicate Violation Notifications**: Made `ViolationMonitor` the single owner of the rich visual "PPE Violation Detected" toast, added report-id dedupe, removed the extra visual toast from `AudioAlert`, and suppressed the generic phone-camera warning when a queued rich violation notice is already shown.
- **[FIXED] Installer Token Replay Regression**: Enforced one-time consumption for `/api/bootstrap/installer` download tokens; replay now returns `403` as required by the provisioning action test.
- **[OPTIMIZED] Low-Egress System Test Defaults**: Changed `deployed_system_conditions_test.py` to default to lightweight deployed endpoint/status checks with `CASM_CONDITIONS_MAX_REPORT_IDS=5`, provider matrix disabled, and `generate-now` probes disabled unless explicitly enabled with `CASM_CONDITIONS_GENERATE_NOW_PROBE=1`.
- **[OPTIMIZED] Low-Egress Frontend Action Test**: Added default image/media/font and report-prefetch blocking to the navigation/timezone action test via `CASM_FRONTEND_ACTION_EGRESS_MINIMAL=1`, reducing Supabase storage image traffic while still exercising the deployed frontend behavior.
- **[VERIFIED] Action/System Gates**: Passed `deployed_system_conditions_test.py`, `deployed_frontend_navigation_timezone_action_test.py`, `deployed_provisioning_action_test.py`, and `deployed_local_failover_sync_contract_test.py` from the project venv. The provisioning suite emitted expected local-only warnings for unconfigured notification channels and fake localhost DB credentials but exited successfully.

### 6. Epic 1 Verification: Local Draft Persistence, Merge, and Reconnect
- **[FIXED] Hard-Refresh Local Report Loss**: Added IndexedDB-backed local report draft persistence in `api.js` so pending local reports keep their original image blob and merge back into Reports/Pending data after a reload or failed cloud fetch.
- **[FIXED] Local/Cloud Report Merge UX**: Local drafts now merge optimistically at the top of cloud/cached report lists with `source_scope=local`, retain a local preview image without pretending an annotated image exists, and are pruned once a synced local/cloud row appears.
- **[FIXED] Live Capture Draft Race**: Local live-camera and uploaded-image violation flows now await local draft persistence before continuing, reducing the chance of losing the pending image if the user refreshes immediately after detection.
- **[FIXED] Reusable Validation Runner**: Updated the local-mode Playwright validation skill runner and docs to the current `Updated_Pipeline_Supabase/tests/...` paths, added `CASM_*` URL overrides, and fixed the reconnect test's local artifact path after the tests moved under `tests/`.
- **[VERIFIED] Local Source Contract**: `CASM_VERCEL_URL=http://127.0.0.1:5000` navigation/timezone action test passed with local draft persistence confirmed after reload (`sourceScope=local`, `hasLocalImageUrl=true`).
- **[VERIFIED] Reconnect/Flapping Contract**: The local reconnect runner passed with routing `cloud -> local -> cloud`; auto sync started and completed after reconnect, and no manual fallback was attempted.
- **[VERIFIED] Deployed Parity and Runtime Contracts**: The deployed parity runner passed report progress parity, local report label badges, cloud runtime health, and NLP metrics contracts. The deployed frontend action test reports the new local-draft helper check as skipped until this commit is deployed, while the same contract passes locally against patched source.

### 7. Partial Local-to-Cloud Handoff for Disappearing Local Backend
- **[FIXED] Unfinished Local Report Handoff**: Added partial artifact handoff for local reports that have `original.jpg`/`annotated.jpg` but no `report.html` yet. Reconnect sync now uploads those image artifacts to Supabase and records a `sync_local_cache_partial` handoff marker instead of waiting forever for the local backend to finish.
- **[FIXED] Cloud Adoption of Local Handoffs**: Updated cloud pending recovery so explicitly handed-off local reports (`sync_local_cache_partial`, `browser_local_draft_handoff`) are eligible for cloud continuation while ordinary local-origin rows remain protected from accidental cloud regeneration.

- **[FIXED] Existing Violation Row Finalization**: Updated the Supabase report generator to update pre-existing partial handoff violation rows with final report HTML/storage keys once cloud generation completes.
- **[OPTIMIZED] Duplicate/Egress Guard**: Added browser handoff in-flight/retry throttling so reconnect races do not repeatedly upload the same IndexedDB draft.
